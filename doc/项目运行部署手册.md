## 1.1 编写目的

  指导开发和测试人员在本地运行部署绩效通系统，完成运行环境与中间件安装配置、
  按顺序启动服务、访问地址与常见故障排查等工作，避免因依赖未就绪导致联调失败。

## 3.1 MySQL 8.0（端口：3306）

主要用于员工账号信息、资料、题库、考核试卷、申诉等数据的持久化存储

### 3.1.1 安装步骤

1. 访问官网地址：https://dev.mysql.com/downloads/installer/

2. 下载安装包，可选两种版本：

     mysql-installer-web-community（在线）

     mysql-installer-community（离线安装包）

3.  启动安装程序，安装类型选择Developer Default（开发者默认），数据库端口保持默认3306

4. 设置root管理员密码

### 3.1.2 数据库初始化

打开CMD / PowerShell，执行命令登录MySQL

```
mysql -u root -p
```

创建数据库，执行建表语句

```
-- 创建数据库
CREATE DATABASE IF NOT EXISTS performance_db DEFAULT CHARSET utf8mb4 COLLATE utf8mb4_general_ci;
USE performance_db;
-- 执行建表与初始化数据脚本（修改为脚本实际存储位置）
SOURCE D:\...\jixiaotong-backend\sql\schema.sql;
```

### 3.1.3 数据库验证

执行以下SQL校验初始化结果：

```
USE performance_db;
SHOW TABLES;
SELECT COUNT(*) FROM sys_user;
```

预期结果：可查看到共14张数据库表，用户表存在10条初始化用户数据。初始化用户数据密码均为123456，登录用户表中为 BCrypt 哈希非明文存储。

## 3.2 Nacos（服务注册与配置中心）—— 端口 8848

### 3.2.1 用途说明

  - 服务发现：gateway-service、performance-service、oa-ai-service 注册到 Nacos
  - 配置中心：主服务拉取 Data ID = performance-service.yaml；网关拉取 gateway-service.yaml
  - 网关通过 lb://performance-service 转发；Feign 通过服务名 oa-ai-service 调用 AI

### 3.2.2 下载部署

  1. 打开 https://nacos.io/download/nacos-server / https://github.com/alibaba/nacos/releases
  2. 推荐 2.3.0，下载 nacos-server-2.3.0.zip
  3. 解压到 如 D:\Env\nacos 路径

### 3.2.3 单机模式启动（CMD / PowerShell）：

```
D:
cd D:\Env\nacos\bin
startup.cmd -m standalone
```

  成功后可在控制台查看类似 Nacos Console started successfully in 520 ms的日志输出

### 3.2.4 服务访问验证

浏览器访问 http://127.0.0.1:8848/nacos

- 默认账号：nacos
- 默认密码：nacos 

### 3.2.5 Nacos配置创建说明

登录控制台 → 【配置管理】 → 【配置列表】 → 【新建配置】

#### 配置一

- Data ID：performance-service.yaml

- Group：DEFAULT_GROUP

- 配置格式：YAML

- 配置内容：完整复制仓库文件 jixiaotong-backend\nacos\performance-service.yaml

  **【注意】修改spring.datasource.password 为本机 MySQL root 密码**

配置成功后点击【发布】

#### 配置二

- Data ID：gateway-service.yaml
- Group：DEFAULT_GROUP
- 配置格式：YAML
- 配置内容：完整复制仓库文件 gateway-service/nacos/gateway-service.yaml

配置成功后点击【发布】

#### 说明

  - spring.config.import 为 optional:nacos:...，未发布配置时可使用本地 application.yml 保障。
    但正式联调建议发布 Nacos 配置，避免不一致。
  - 关闭 Nacos 窗口即停止服务；开发和测试时需保证nacos正常运行。


3.3 Redis（端口：6379）
--------------------------------------------------------------------------------
### 3.3.1 用途说明

  - 用户登录凭证：保存登录生成的 JWT 令牌，借助Redis实现登录状态维持。Redis 未启动时，用户无法登录
  - 资料列表数据缓存：缓存资料列表信息，减轻数据库查询压力，缓存有效期 2 小时。
  - 业务临时数据缓存：用于保存提交锁、草稿内容、切屏检测计数等临时业务数据。

### 3.3.2 安装

#### 方法一  下载压缩包

**下载安装**

Windows版Redis下载地址： https://github.com/redis-windows/redis-windows/releases

推荐版本：7.2.x，下载压缩包

解压到例如 D:\Env\Redis

**启动**

打开 CMD，进入 Redis 目录执行：

```
redis-server.exe
```

**验证**

新CMD窗口执行

```
redis-cli
ping
```

返回PONG表示服务正常启动

#### 方法二  Docker 部署 Redis

cmd执行命令后台启动 Redis 容器：

```
docker run -d -p 6379:6379 --name redis redis:7
```

**验证**

打开CMD执行

```
redis-cli
ping
```

返回PONG表示服务正常启动

**【注意事项】**

- 本项目默认无 Redis 密码
- 如果自行修改配置 密码认证，需要同步更新 Nacos 内 `performance-service.yaml`、`gateway-service.yaml` 的 Redis 连接配置


3.4  Elasticsearch（历年绩效归档全文检索）—— 端口 9200

### **3.4.1 用途说明**

  经理端「历年绩效 ES 检索」关键词 + 分数区间搜索。

### 3.4.2 下载

   官方网站地址：https://www.elastic.co/cn/downloads/elasticsearch

```

```

### 3.4.4 启动

CMD 执行命令：

```
D:
cd D:\Env\elasticsearch\bin
elasticsearch.bat
```

### 3.4.5 验证

浏览器访问 http://localhost:9200/，预期返回含 cluster_name、version、tagline 的 JSON。

### 3.4.6 内存提示

八、常见报错排查方案
--------------------------------------------------------------------------------

## 8.1 Nacos 相关

### 8.1.1 启动后端报 Connection refused / Unable to connect to Nacos

  原因：Nacos 未启动或非 standalone 启动失败

  处理：
    1. 确认用 startup.cmd -m standalone
    2. 访问 http://localhost:8848/nacos
    3. 检查防火墙是否拦截 8848

### 8.1.2 User nacos not found / login failed for user nacos

  原因：客户端 application.yml 配置了 nacos.username/password，但本地 Nacos未启用鉴权用户体系

  处理：删除客户端中的 username、password 配置后重启后端

### 8.1.3 服务列表看不到实例

  原因：(1) 后端未启动成功   (2) discovery.enabled=false  (3) 连错 server-addr

  处理：确认 application.yml 中 server-addr=127.0.0.1:8848 且 enabled=true

### 8.1.4 com.alibaba.nacos.api.exception.NacosException

  出错提示：com.alibaba.nacos.api.exception.NacosException: Code: 401, Message: User not found!

  原因：Nacos 开启账号认证，配置中缺少账号密码

  处理：确保网关、业务后端和AI 服务配置内，`discovery` 和 `config` 需要分别配置账号信息

## 8.2 Redis 相关

### 8.2.1 登录接口失败

  出错提示： Unable to connect to Redis / Connection refused:6379

  原因：Redis 未启动

  处理：先启动 Redis再登录

### 8.2.2  登录成功刷新后 401 / 提示登录已失效

  原因：Token 未带、Redis 被清空或旧前端缓存无 token
  处理：

1. 清除浏览器 localStorage 后重新登录
2. 确认请求头含 Authorization: Bearer
3. redis-cli 中查看是否存在 auth:token:* 键

## 8.3 MySQL 相关

### 8.3.1 Access denied for user 'root'@'localhost'

  原因：密码与配置不一致

  处理：修改 Nacos 中 datasource.password 与本地 application.yml，保持一致后重启

### 8.3.2 Unknown database 'performance_db'

  原因：未建立数据库

  处理：CREATE DATABASE performance_db ... 后重新导入 schema.sql

### 8.3.3 Table 'xxx' doesn't exist

  原因：未执行 schema.sql 或执行到一半失败

  处理：重新完整执行 schema.sql

### 8.3.4 时区相关警告或时间偏差

  处理：连接串已含 serverTimezone=Asia/Shanghai，保持不变即可

## 8.4 Elasticsearch 相关

### 8.4.1 后端启动慢 / 日志连接 9200 失败

  原因：ES 未启动或安全认证开启

  处理：

1. 启动 ES 并关闭 xpack.security
2. 检索接口会降级 MySQL，一般不阻断主流程；可先保证登录与答题

###  8.4.2  检索无数据

  原因：索引为空，尚未同步

  处理：使用系统提供的归档同步/检索接口，或先完成若干考核归档后再查

## 8.5 后端 （网关 / Feign / AI 相关）

###  8.5.1  端口被占用 Address already in use: 8080/8081/8088

  处理：

- 终止进程

  ```
  netstat -ano | findstr ":8080 :8081 :8088"
  taskkill /PID <pid> /F
  ```


-   修改 server.port 后同步改前端 vite proxy（网关默认 8088）

###  8.5.2 经网关访问失败

  出错提示：503 / Unable to find instance for performance-service

  原因：主服务未注册到 Nacos，或网关早于主服务启动

  处理：先启动 performance-service，确认 Nacos 有实例后再启/重启 gateway-service

###  8.5.3 主观题长时间「生成中」或评语为降级文案

  原因：oa-ai-service 未启动、未注册到 Nacos或 Ollama 未就绪

  处理：

1.  确认 8081 已启动且 Nacos 有 oa-ai-service
2. 可选启动 Ollama 并 pull 模型
3. 未开 Ollama 时依赖 fallback，应仍能得到启发式评语；若完全失败查看主服务日志 Feign 超时

###  8.5.4 Feign 超时（readTimeout）

  原因：大模型推理慢

  处理：配置已将 oa-ai-service readTimeout=120000ms，可换更小模型或先用降级

###  8.5.5 登录返回 429

  原因：同一 IP 短时间登录次数超过网关限流（默认60秒20次）

  处理：稍后再试，或调整 gateway.login-rate-limit 后重启网关

## 8.6 前端相关

###  8.6.1 页面打开但接口 Network Error / 404

  原因：网关或主后端未启动，或未用 npm run dev启动前端

  处理：确保 8088（网关）与 8080（主服务）已启动，并用 Vite 开发服务器访问 5173

###  8.6.2 登录后白屏或跳转异常

  原因：角色字段异常或路由未匹配

  处理：确认库中 role 为 ADMIN/MANAGER/EMPLOYEE；清理缓存重登

###  8.6.3 npm install 失败

  处理：切换镜像后重试：
    npm config set registry https://registry.npmmirror.com
    npm install
